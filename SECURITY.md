# Security Policy

## Reporting A Vulnerability

Please do not open a public issue for security-sensitive reports.

Email the maintainer with:

- A short description of the issue.
- Steps to reproduce.
- Any affected route, API endpoint, or browser flow.
- The impact you believe it may have.

If you do not already have a preferred contact path, open a minimal public issue
asking for a private security contact without including exploit details.

## Scope

Security reports are most useful when they involve:

- Exposure of secrets or credentials.
- Unauthorized access to internal health or operational endpoints.
- Client-side vulnerabilities that could affect users of `puca.dev`.
- Data handling issues involving location, favorites, or local browser storage.

Púca does not have user accounts, server-side user profiles, ticketing, or
payment flows.
